# Гайд по настройке: скины и внутриигровые покупки (Яны)

Краткая пошаговая настройка скинов игрока и покупок через Яндекс Игры (Яны).

---

## 1. Каталог товаров в Яндекс Играх

В [кабинете разработчика](https://console.yandex.ru/) → ваша игра → **Внутриигровые покупки** создайте три товара:

| ID товара   | Название (для отображения) | Цена (Яны) |
|-------------|----------------------------|------------|
| `skin_scarf`| Скин Scarf                 | 49         |
| `skin_ninja`| Скин Ninja                 | 99         |
| `skin_gold` | Скин Gold                  | 149        |

**Важно:** ID должны совпадать с константами в коде (`SkinPurchaseHandler.SkinIdScarf` и т.д.).

---

## 2. Симуляция покупок в Unity Editor (опционально)

Чтобы тестировать получение каталога в редакторе:

1. **Plugin Your Games** → настройки модуля **Payments** (или объект **YG2** в сцене).
2. В массиве **Purchases** добавьте три элемента с теми же **ID**: `skin_scarf`, `skin_ninja`, `skin_gold`.

Подробнее: [документация Payments — симуляция](https://max-games.ru/plugin-yg/doc/pay/).

---

## 3. Префабы скинов

У вас есть три меша/модели скинов (scarf, ninja, gold). Для каждого:

1. Создайте **префаб** (перетащите объект из иерархии в папку проекта).
2. На префабе должен быть **Animator** с тем же набором параметров, что у текущего персонажа (Speed, isGrounded, IsTaking и т.д.), и тот же Animator Controller (или совместимый).
3. Сохраните префабы (например, в `Assets/Assets/Player/` или `Assets/Assets/Prefabs/Skins/`).

---

## 4. SkinManager и SkinPurchaseHandler

**SkinManager** — единая точка контроля скинов (какой надет, какие куплены, применение на персонажа).

1. Создайте в сцене объект (например, **SkinManager**).
2. Добавьте компонент **SkinManager**. Заполните:
   - **Player** — объект с **ThirdPersonController**.
   - **Default Skin Prefab** — префаб скина Obby (по желанию; если пусто, при выборе «дефолт» модель не меняется).
   - **Scarf / Ninja / Gold Skin Prefab** — префабы скинов.

**SkinPurchaseHandler** — только подписка на события покупки YG2 и консумирование.

3. На тот же или другой объект добавьте **SkinPurchaseHandler** (автоматически добавится **ConsumePurchasesYG**). Ссылки на игрока и префабы не нужны — всё берётся из SkinManager.

---

## 5. Кнопка SKIN и модальное окно скинов

1. **Кнопка SKIN**  
   На кнопку добавьте **OpenSkinModalButton**. В инспекторе укажите:
   - **Modal Window** — объект модального окна (родитель ButtonsUI) ИЛИ
   - **Modal Controller** — объект с **SkinModalController** (предпочтительно).

2. **SkinModalController** (на объекте модального окна или рядом):
   - **Modal Window** — родитель ButtonsUI (само модальное окно; по умолчанию выключен).
   - **Blur Overlay** — полноэкранная панель для блюра/затемнения (Canvas → Panel на весь экран, Image с полупрозрачным чёрным или материал с блюром; по умолчанию выключена). В иерархии должна быть **под** модальным окном, чтобы модалка была поверх.
   - Клик по оверлею закрывает модалку (на Overlay можно повесить **Button** без картинки или скрипт сам добавит обработчик клика).

3. **Слоты Obby, Scarf, Ninja, Gold**  
   В каждом родителе (Obby, Scarf, Ninja, Gold) на кнопку добавьте **SkinSlotUI**:
   - **Slot** — Obby / Scarf / Ninja / Gold.
   - **Button** — кнопка (если не задана — берётся с этого объекта).
   - **Button Label** — TextMeshProUGUI с текстом (TAKEN/ВЗЯТ, BUY/КУПИТЬ, PICK/ВЫБРАТЬ — подставляется автоматически по локализации и состоянию скина).
   - **Modal Controller** (опционально) — чтобы после «ВЫБРАТЬ» модалка закрывалась.

Тексты кнопок: **взят** → «ВЗЯТ»/«TAKEN»; **не куплен** → «КУПИТЬ»/«BUY»; **куплен, не надет** → «ВЫБРАТЬ»/«PICK».

---

## 6. Меши скинов в мире (BuyableSkin)

На каждый **меш скина** в иерархии (витрина/стенд), с которым игрок взаимодействует:

1. Добавьте компонент **BuyableSkin** (наследник InteractableObject).
2. Настройте:
   - **Product Id** — один из: `skin_scarf`, `skin_ninja`, `skin_gold`.
   - **Purchase Handler** — можно оставить пустым; тогда в сцене автоматически ищется **SkinPurchaseHandler**.
3. Заполните остальное как у любого интерактивного объекта:
   - **UI Prefab** — префаб подсказки с кольцом прогресса (как у других InteractableObject).
   - При необходимости: радиус, время удержания, точка взаимодействия.

Текст подсказки будет **«Купить» / «Buy»** в зависимости от локализации.

---

## 7. Проверка

- **Запуск игры:** при старте вызывается консумирование (ConsumePurchasesYG), затем применяется сохранённый скин из сохранений.
- **Покупка:** подойти к мешу скина → удержать E (или тап) → откроется окно покупки Яндекс Игры → после успешной оплаты скин применится и сохранится.
- **После перезагрузки:** выбранный скин должен подставиться автоматически.

---

## 8. Сброс скина

При полной очистке сохранений через **GameStorage** (Clear Storage) поле **currentSkinId** сбрасывается — скин возвращается к виду по умолчанию (модель, которая была на игроке в сцене до применения скинов).
